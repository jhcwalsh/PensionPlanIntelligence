# Digital Twin v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fill the twin's empty facets — structured IPS parsing, CAFR actuarial/funding extraction, canonical asset classes, and a manager-roster reconciliation table — and surface them in the twin.

**Architecture:** Two new LLM extractors clone `extract_cafr_investments.py`'s tool-forced-JSON house pattern (cached system prompt, `tool_choice`, tenacity retry, `text_hash` + `prompt_version` skip) into new tables (`ips_extract`+`ips_allocation`, `cafr_actuarial`). A Haiku-assisted asset-class mapping mirrors `scripts/normalize_managers.py` into `data/asset_class_mappings.json`. A deterministic roster job materializes `plan_manager_roster`. `twin_builder.py` bumps to `twin_v1` and consumes all four: policy gains IPS structure, allocation gains canonical labels + IPS targets, governance gains IPS consultant + actuary firm, funding_actuarial becomes real.

**Tech Stack:** anthropic SDK (`claude-sonnet-4-6` extraction, Haiku classification), SQLAlchemy + idempotent `init_db()`, pymupdf section location.

## Global Constraints

- Never ALTER TABLE; new SQLAlchemy classes + `init_db()` only. All new tables rebuildable from sources (replace-on-reextract, like `save_extract`).
- Every extractor honors `LLM_MODE=mock` (already set autouse in tests/conftest.py): no API call, return the canned payload defined in its task. Production model: `MODEL = "claude-sonnet-4-6"`.
- Skip logic everywhere: `text_hash` (md5 of input text) + `PROMPT_VERSION` string on the row; unchanged → skip.
- Raw labels always kept beside canonical; unknown labels map to `"unmapped"`, never dropped. Canonical taxonomy is the `ASSET_CLASS_CANONICAL` constant (Task 1) — the classifier must never invent values outside it.
- `twin_builder` stays LLM-free; it only reads the new tables/mapping file.
- Tests: `LLM_MODE=mock pytest tests/ -q`, `tmp_db` isolation; never stage `db/pension.db` in task commits (coordinator commits it after the backfill).
- Do not `git push`. Commit messages end with the repo's standard Claude co-author trailer.
- `TWIN_SCHEMA_VERSION` becomes `"twin_v1"` (Task 6). v0 JSON keys never change meaning — v1 only adds keys.

## File Structure

- `database.py` — `IpsExtract`, `IpsAllocation`, `CafrActuarial`, `PlanManagerRoster`, `ASSET_CLASS_CANONICAL` (Task 1).
- `data/asset_class_mappings.json` + `scripts/normalize_asset_classes.py` + loader in `twin_builder.py` (Task 2).
- `extract_ips.py` (Task 3), `extract_cafr_actuarial.py` + locate-helper parametrization in `extract_cafr_investments.py` (Task 4).
- `scripts/build_manager_roster.py` (Task 5).
